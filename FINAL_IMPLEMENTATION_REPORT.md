## üé® Naming Convention Standards

### Components
- ‚úÖ **PascalCase**: `BookingSheet.tsx`, `ServiceCard.tsx`
- ‚úÖ **File name matches component name**
- ‚úÖ **Props interface**: `{ComponentName}Props`

### Variables & Functions
- ‚úÖ **camelCase**: `userName`, `getData()`, `handleSubmit()`
- ‚ùå **No snake_case**: `user_name`, `get_data()`
- ‚ùå **No kebab-case**: `user-name`, `get-data()`

### Constants
- ‚úÖ **UPPER_CASE**: `API_BASE_URL`, `MAX_RETRIES`

### Interfaces & Types
- ‚úÖ **PascalCase**: `UserInterface`, `BookingType`
- ‚úÖ Optional `I` prefix: `IUser` (optional standard)

### Exceptions (by design)
- ‚úÖ **shadcn/ui components**: `alert-dialog.tsx`, `date-picker.tsx`
- ‚úÖ **Config files**: `vite.config.ts`, `tailwind.config.ts`
- ‚úÖ **Test files**: `*.test.tsx`, `*.spec.tsx`

## üí° Recommendations for AI Agents

### When Creating New Components

1. **Use PascalCase for component names**
   ```typescript
   // ‚úÖ Good
   export const ServiceCard = ({ service }) => { ... };
   export function BookingSheet() { ... };

   // ‚ùå Bad
   export const service_card = () => { ... };
   export default function component() { ... };
   ```

2. **Name file after component**
   ```bash
   # ‚úÖ Good
   ServiceCard.tsx ‚Üí export const ServiceCard

   # ‚ùå Bad
   serviceCard.tsx ‚Üí export const ServiceCard
   ```

3. **Use descriptive names**
   ```typescript
   // ‚úÖ Good
   BookingSheet, UserProfile, ServiceCard

   // ‚ùå Bad
   Component1, TempComponent, Stuff
   ```

### When Reviewing Code

1. **Check naming conventions**
   - File names in PascalCase
   - Component exports match file names
   - No console.log statements
   - Props interfaces properly named

2. **Run quality checks**
   ```bash
   npm run lint     # Check naming
   npm run type-check  # Check types
   ```
